[SIZE=4][B]Tracking the Cause of Lag[/B][/SIZE]
[SIZE=3]EzClean makes it easy to diagnose and track down the source of server lag, especially when it is related to scheduled tasks or plugin activity. Use the following features to pinpoint lag causes:[/SIZE]
[LIST]
[*][B]Scheduler Usage Snapshots:[/B] Run `[icode]/ezclean usage[/icode]` to instantly view a breakdown of all pending sync and async tasks, sorted by plugin. This helps you identify which plugins are scheduling the most work and may be contributing to lag.
[*][B]Live Usage Monitoring:[/B] Use `[icode]/ezclean usage live[/icode]` to start a real-time feed in chat and action bar, showing task counts and the busiest plugins as they change. This is ideal for catching lag spikes as they happen and correlating them with plugin activity.
[*][B]Plugin Filtering:[/B] Add a plugin name to `[icode]/ezclean usage[/icode]` or `[icode]/ezclean usage live[/icode]` to focus on a specific plugin's scheduler activity.
[*][B]Stopping the Feed:[/B] Use `[icode]/ezclean usage stop[/icode]` to end a live monitoring session at any time.
[*][B]Visual Feedback:[/B] The action bar graph and chat output make it easy to spot sudden increases in scheduled tasks, which often correlate with lag events.
[/LIST]
[SIZE=2][I]Tip: If you notice a spike in scheduled tasks or a particular plugin dominating the scheduler, investigate that plugin's configuration or recent activity. EzClean's tools are especially useful during peak times or after adding new plugins.[/I][/SIZE]

[SIZE=4][B]Advanced Death Chest Options (Optional)[/B][/SIZE]
[LIST]
[*][B]Per-player chest limits[/B] – Restrict the number of active death chests each player can have at once to prevent abuse and clutter.
    [spoiler="Example: Limit to 2 chests per player"]
    [code=yml]
    max-chests-per-player: 2
    [/code]
    [I]Players exceeding the limit will have their oldest chest removed automatically.[/I]
    [I]Usage:[/I] [I]Set this value in death-chests.yml to control how many chests each player can have at once. Useful for preventing clutter and abuse on busy servers.[/I]
  [I]Permission:[/I] [icode]ezclean.deathchest.limit.bypass[/icode] lets a player ignore the chest limit.[/spoiler]
[*][B]Configurable loot protection[/B] – Only the owner can open their death chest for a configurable time after death (e.g., 5 minutes), after which anyone may access it.
    [spoiler="Example: 5-minute owner protection"]

    loot-protection:

    [/code]
    [I]Prevents looting by others until the timer expires. Great for PvP servers![/I]
    [I]Usage:[/I] [I]Enable this to give players a grace period to recover their items before others can access the chest. Adjust the timer as needed for your server's play style.[/I]
  [I]Permission:[/I] [icode]ezclean.deathchest.protection.bypass[/icode] lets a player open any chest regardless of protection.[/spoiler]
[*][B]Hologram or particle effects above chests[/B] – Display a floating label or visual effect above each death chest for easy spotting.

      text: "<gold>Death Chest</gold>"
    particles:
      enabled: true
      type: VILLAGER_HAPPY
    [/code]
    [I]Customizable text and particle type. Helps players find their chests quickly.[/I]
    [I]Usage:[/I] [I]Set these options to make death chests more visible. Holograms and particles can be themed to match your server and help players locate their chests easily.[/I]

[SIZE=4][B]Admin Commands[/B][/SIZE]
[LIST]
[*][ICODE]/ezclean run [cleaner][/ICODE] – Trigger any configured profile instantly when staff need a manual sweep.
[*][ICODE]/ezclean cancel [cleaner][/ICODE] – Delay the next cleanup cycle (and optionally charge a Vault fee) via command or the clickable broadcast prompt.
[*][ICODE]/ezclean time [cleaner][/ICODE] – Check how many minutes remain before the next scheduled cleanup runs.
[*][ICODE]/ezclean reload[/ICODE] – Reload EzClean's configuration bundle without restarting to apply new profiles, messages, or entity rules.
[*][ICODE]/ezclean usage [plugin|live|stop] [plugin][/ICODE] – Diagnose heavy scheduler usage, start a live monitor, or cancel an active session (`ezclean.usage`).
[*][ICODE]/ezclean stats [cleaner][/ICODE] – View detailed statistics for a cleaner profile, including run counts, total entities removed, average duration, TPS impact, and top affected worlds/groups.
[*][ICODE]ezclean.deathchest.limit.bypass[/ICODE] – Allows a player to ignore the per-player death chest limit.
[*][ICODE]ezclean.deathchest.protection.bypass[/ICODE] – Allows a player to open any death chest regardless of loot protection timer.
[/LIST]

[SIZE=4][B]Quick Start[/B][/SIZE]
[LIST]
[*]Drop `EzClean.jar` into your Paper or Purpur `plugins/` directory and restart the server.
[*]Let EzClean generate `cleaners/default.yml`, `messages.yml`, and `death-chests.yml` inside its data folder on first launch.
[*]Duplicate `cleaners/default.yml` for world-specific schedules, then adjust world lists, removal toggles, broadcast cadence, and cancel costs per profile.
[*]Theme broadcasts in `messages.yml`, tweak death chest timers in `death-chests.yml`, and reload the plugin (or restart) to apply changes before checking `/ezclean time`.
[/LIST]

[SIZE=4][B]WorldGuard Bypass Tag[/B][/SIZE]
[LIST]
[*]Install [URL='https://enginehub.org/worldguard']WorldGuard[/URL] alongside EzClean to unlock region-based exclusions.
[*]EzClean automatically registers the `[icode]ezclean-bypass[/icode]` state flag during startup—no manual flag setup required.
[*]Use `[icode]/rg flag <region> ezclean-bypass allow[/icode]` to mark safe zones (spawn, showcases, redstone labs) that should never be swept.
[*]Switch the flag back to `[icode]deny[/icode]` (or remove it) when you want EzClean to resume removing entities from that region.
[*]Entities inside an `[icode]allow[/icode]` region are skipped while the rest of the server is tidied normally.
[/LIST]
